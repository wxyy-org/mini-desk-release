# Mini Desk

一款 macOS 菜单栏工具，可在桌面图标层级为每块显示器嵌入全屏网页窗口，适合用作桌面信息看板、本地开发预览、实时仪表盘等场景。

## 功能特性

- **多显示器支持** — 自动检测已连接的显示器，每块物理显示器使用基于 EDID 的稳定硬件标识，插拔或重启后配置仍绑定同一台显示器
- **连接池** — 曾连接过的显示器保留在「连接池」中，断开后仍可编辑 URL，可随时移除不再需要的条目
- **桌面级窗口** — 网页窗口位于桌面图标层级之上、普通窗口之下，不干扰日常操作；支持鼠标与键盘输入
- **识别显示器** — 点击配置卡片上的瞄准镜图标，对应屏幕底部会短暂显示高亮浮层，便于区分正在配置的是哪一块屏
- **菜单栏常驻** — 通过状态栏图标快速访问设置、刷新、退出等操作
- **热插拔与睡眠** — 监听显示器变化与锁屏/唤醒；WebView 在锁屏时挂起加载与重试，减少无效请求
- **窗口生命周期** — 桌面网页窗口创建后不销毁（仅隐藏），降低 WebKit 与窗口动画相关的极端场景风险
- **缓存管理** — 支持全局或单屏清空 WebKit 缓存并强制刷新页面
- **国际化** — 支持中文和英文界面，可在设置中动态切换语言
- **URL 连通性验证** — 启用屏幕前自动测试 URL，也可手动触发测试
- **自动刷新 / 重试** — 可配置页面定时自动刷新，以及加载失败后的自动重试（含上限，避免死循环）
- **开机自启** — 支持 macOS 原生的登录项管理
- **内存管理** — 实时内存仪表，支持按屏幕冻结/解冻（快照替换 WebView 释放内存）；可配置自动冻结策略（超阈值或系统内存压力时自动冻结后台屏幕）；一键清除站点数据
- **持久化诊断日志** — 运行问题可查看 ~/Library/Logs/MiniDesk/MiniDesk-YYYY-MM-DD.log（最多保留 7 天）

## 系统要求

- macOS 14.0 (Sonoma) 或更高版本

## 快速开始

1. 下载并打开 Mini Desk.dmg，将左侧 **Mini Desk.app** 拖入右侧 **Applications（应用程序）**  alias
2. 启动应用后，菜单栏会出现 Mini Desk 图标
3. 点击图标 → 设置（⌘,）打开配置面板
4. 在左侧选择 Displays，为当前已连接的显示器填入网页 URL
5. 打开「在此屏幕显示网页桌面」开关（会先验证 URL），通过后桌面网页即可显示
6. 若有多块屏分不清，点击卡片右侧识别图标（瞄准镜）确认物理屏幕

## 设置面板

左侧为 Sidebar（约 200pt 宽），主区域为详情：

| 标签 | 说明 |
|------|------|
| General | 语言、开机自启、默认 URL、加载超时、自动刷新、失败重试等 |
| Displays | 当前已连接的显示器及其 URL / 启用开关 |
| Connection Pool | 曾连接、当前未连接的显示器配置；可编辑或从池中移除 |
| Memory | 内存监控仪表、按屏幕冻结/解冻、自动冻结策略、站点数据清理 |
| About | 版本与作者信息 |

## 菜单栏操作

| 菜单项 | 快捷键 | 说明 |
|--------|--------|------|
| 显示窗口 | — | 将所有桌面网页窗口置前 |
| 设置 | ⌘, | 打开设置面板 |
| 清空缓存并刷新 | ⌘R | 清除 WebKit 缓存并重新加载所有页面 |
| Web 检查器 | F12 | 切换当前聚焦屏幕的 WebKit 开发者工具 |
| 退出 | ⌘Q | 退出应用 |

## 技术栈

- Swift 5 + SwiftUI + AppKit
- WebKit (WKWebView)
- Xcode String Catalog 本地化
- UserDefaults JSON 持久化
- ServiceManagement (SMAppService) 登录项
- 零外部依赖
